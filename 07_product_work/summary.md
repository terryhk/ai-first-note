# 本章小结

### 📊 概率工具箱

回到全书的核心框架：行为 × 概率 = 结果。这一章的每一条工作线，你的"行为"都是把流程结构化，从而提升AI输出"成功率"的概率。

| 场景 | 无方法 P | 有方法 P | 方法 |
|------|---------|----------|------|
| 写周报 | ≈40% | ≈80%+ | 四步法：提取→归纳→成文→提醒 |
| 改PPT | ≈50% | ≈85%+ | 受众画像+结构化重写+一页一结论 |
| 发邮件 | ≈20% | ≈80%+ | 先提炼行动项再成文 |
| 规范对比 | ≈30% | ≈75%+ | 结构化对比表 |

数据不会说谎。每一条工作线的效率提升，都是因为**模板把P的起点拉高了**——你不需要每次从零开始。就像搬东西上楼：没有电梯，你扛上去，每次都很累；有了电梯，你只需要按楼层按钮。电梯不会帮你打包，但爬楼这个最消耗体力的环节，它替你解决了。

四步法、结构化对比表、一页一结论——这些都是你的"电梯"。它们不替代你的思考，但把重复劳动这个最浪费精力的环节自动化了。

### 边际成本与机会成本分析

产品工作流的经济账很清晰：**一次性固化模板的投入，换来后续每次使用的边际成本趋近于零。**

不固化模板的机会成本是什么？是你每周都在重复"找资料、对口径、补缺口"——随着事项增多，这是典型的规模不经济：你负责的产品线越多、需要汇报的事项越多，每周花的重复劳动时间就越多，而且线性增长，没有天花板。

来看看周报的经济账：

| | 不自动化 | 自动化 |
|------|---------|--------|
| 初始投入 | 0小时 | ~3小时（搭建模板+调试流程） |
| 每周耗时 | 2-3小时 | 30分钟审校 |
| 月均节省 | — | 6-9小时 |
| 年均节省 | — | 70-100小时 |

每月省6-9个小时，每年省70-100个小时。这些时间不是"加班的时间"，是你本该用于思考、判断、做决策的时间——但以前都被重复劳动挤占了。

更深一层的隐性成本：如果你不把规范对比做成对照表、不把风险分析做成风险处置卡，每次都可能漏项——漏项的代价不是时间，是返工、延期或合规事故。这种隐性机会成本往往比显性时间成本大一个数量级。一次合规事故的成本，可能是你一整年省下的时间都补不回来的。

所以，固化模板的经济逻辑不仅仅是"省时间"，更是"降风险"——降低你在重复劳动中出错的概率。回看概率工具箱：无方法写周报P≈40%，有方法P≈80%+。40%的成功率意味着你有60%的概率出问题——漏信息、口径错、格式乱。80%的成功率意味着你只有20%的概率需要手动修正。从60%出错降到20%出错，这个收益远不止"省了两个小时"——它还包含了"减少了一次被领导质疑的机会"、"减少了一次信息遗漏引发的返工"、"减少了周五晚上加班赶周报的痛苦"。这些隐性收益加在一起，远比表面看到的大。

### 📋 可复用模板

下面五个模板，是我日常反复使用的。直接复制，填入你自己的内容即可。

#### 模板一：「周报四段式」

```
角色：你是我周报的编辑助理。
任务：根据我提供的本周素材，按四段式生成周报初稿。
输出格式：
1. 【本周事实】：关键数据与完成事项
2. 【进展】：各事项的推进状态
3. 【风险】：当前存在的风险点及程度（关注/预警/紧急）
4. 【下周计划】：下周重点事项与预期目标

约束：
- 只从我提供的素材中提取，不编造未提及的信息
- 数据口径必须与原料一致，不得自行合并或拆分
- 风险程度必须准确标注，不得将"可能"说成"确定"
```

#### 模板二：「规范对比表」

```
角色：你是一名业务规范分析师。
任务：对比我提供的两份规范文件，生成结构化对比表。
输出格式：
| 章节 | 旧规范要求 | 新规范要求 | 差异 | 建议处置 | 优先级 |

约束：
- 逐章对比，不遗漏任何实质性差异
- 差异描述必须具体，不得用"略有不同"等模糊表述
- 建议处置只给方向，不做最终判断
- 优先级留空，由我填写
```

#### 模板三：「一页一结论PPT」

```
角色：你是一名商业表达顾问。
任务：把我提供的技术方案，转化为PPT初稿。
输出格式：每页PPT包含：
1. 结论（一句话）
2. 证据（2-3条）
3. 影响（1-2句）
4. 下一步（1条行动项）

受众：[填写：老板/客户/研发]
受众关注：[填写：结论与风险/价值与案例/技术细节与边界]

约束：
- 每页只讲一个结论，不得合并多个论点
- 语言适配受众，不得在老板版里写技术细节，不得在研发版里写商业术语
```

#### 模板四：「会议纪要5行版」

```
角色：你是一名会议纪要助手。
任务：根据我提供的会议内容，生成5行版会议纪要。
输出格式：
1. 结论：本次会议达成的核心共识
2. 行动项：接下来要完成的具体事项
3. 负责人：每个行动项的Owner
4. 截止时间：每个行动项的Deadline
5. 下次会议：时间与议题

约束：
- 结论不超过1句话
- 行动项必须是具体可执行的，不得写"继续推进"这种空话
- 每行不超过2句话，总计不超过10句话
```

#### 模板五：「技术建议邮件」

```
角色：你是一名技术顾问邮件助手。
任务：根据我提供的信息，生成技术建议邮件。
输出格式：
1. 一句话结论
2. 三条支撑证据
3. 两步落地计划

约束：
- 结论放在邮件开头，不得先写背景再出结论
- 每条证据不超过2句话
- 落地计划必须具体，包含"做什么"和"什么时候"
- 语气专业但不生硬，像跟同事沟通而不是写论文
```

---

这五个模板的用法都一样：复制到你的AI工具里，替换方括号里的内容，跑一遍，审校输出。刚开始你可能需要两三次调试才能让AI的输出符合你的预期——前两小时是投入期。但一旦调好了，后面的每一周，你只需要填素材、跑模板、审校三步。从"每次两小时"到"每次三十分钟"，差额就是你赢得的思考时间。

这，就是公理一最朴素的体现：把体力型重复劳动交给AI，把稀缺的判断力留在你自己手里。
