# 8.3 ReAct：Agent的"大脑"

Skill让Agent有能力，Harness让Agent能跑起来。那Agent自己怎么决定"该做哪一步"？靠ReAct。

ReAct这个词来自一篇2022年的论文"ReAct: Synergizing Reasoning and Acting in Language Models"。它的核心思想极其简单，但极其强大：

**思考（Reason）→ 行动（Act）→ 观察（Observe）→ 再思考 → 再行动 → ……**

就这么循环下去，直到任务完成。

你在第4章已经见过ReAct这个名字了。那里我把它作为"思维链的延伸"简单介绍了一下——思维链让AI"想清楚再输出"，ReAct让AI"想一步、做一步、看结果、再想下一步"。现在我们把它放进Agent的场景里，看看它怎么让Agent自己跑流程。

拿我的行业简报Agent举例：

**第一轮：**
- 思考："我需要搜索本周AI行业动态。关键词：AI、LLM、Agent。"
- 行动：调用搜索Skill
- 观察：收到10条结果

**第二轮：**
- 思考："其中3条是广告，2条是旧闻，剩下5条有价值。这5条中有2条技术相关、3条商业相关。"
- 行动：调用摘要Skill，分别摘要5条新闻
- 观察：5条新闻的摘要已生成

**第三轮：**
- 思考："5条摘要已就绪。按技术/商业分类排版，生成简报邮件格式。"
- 行动：调用邮件排版Skill
- 观察：简报邮件初稿已生成

**第四轮：**
- 思考："初稿完成，需要发送给Terry审批。"
- 行动：调用发邮件Skill，发送到我的邮箱
- 观察：邮件已发送

四轮循环，Agent自己完成了"搜→筛→摘要→排版→发送"的全流程。我在干什么？我在喝咖啡——直到邮件弹出来，我审校一下，确认无误，这事就算完了。

ReAct和纯思维链的区别在哪里？思维链是"想完再做"，ReAct是"想一步做一步"。为什么Agent必须用ReAct而不是思维链？因为Agent面对的是真实的世界——搜索会返回意外结果，API可能会报错，信息源可能变化。如果Agent想完再做，它没法应对中间的意外；如果它想一步做一步，就能根据每一步的观察调整策略。就像开车——你不会在出发前把每个转弯的方向盘角度都算好再上路，你是一边开一边看路标一边调整方向。ReAct就是Agent的"边开边看"。

判断原则：**如果你的任务是一步到位的，用思维链就够了；如果你的任务需要多步、每步依赖上一步的结果，就必须用ReAct。** 行业简报是典型的多步任务——搜完才知道有多少条、筛完才知道剩几条、摘完才知道怎么分类——所以必须用ReAct。而一次性的写邮件、改PPT，思维链就够了。
