# 8.4 你的第一个Agent任务

了解了Skill、Harness、ReAct三个部件，现在把它们串起来，设计你的第一个Agent任务。

我的第一个Agent任务就是前面说的"AI行业动态简报"。我用的工具是OpenClaw，流程长这样：

**第一步：定义目标。**

"每周一早上9点，自动生成一份AI行业动态简报，发送到我的邮箱。"

目标要具体。不要说"帮我关注行业动态"——太模糊了，Agent不知道"关注"要做到什么程度。要说清楚：输出是什么、格式是什么、给谁、什么时候要。

**第二步：拆解SOP。**

把流程拆成Agent能执行的步骤——就像给实习生写工作手册：

1. 搜索：用关键词"AI、LLM、Agent、大模型"搜索本周新闻
2. 过滤：筛掉广告、旧闻、不相关的内容
3. 摘要：为每条有效新闻写50字以内的摘要
4. 分类：按"技术/商业/政策"归类
5. 排版：按简报模板格式化
6. 发送：将简报发送到指定邮箱

每一步都是ReAct循环中的一轮"行动"。Agent思考"我该做什么"→执行对应步骤→观察结果→思考"下一步做什么"。

**第三步：配置Skill。**

每一步对应一个或多个已有Skill：

- 搜索步骤 → 用"web搜索"Skill
- 过滤和摘要步骤 → 用"文本摘要"Skill
- 分类和排版步骤 → 用"模板生成"Skill
- 发送步骤 → 用"邮件"Skill

大多数Harness平台（包括OpenClaw）都预装了常用Skill，你不需要从零写。就像新手机出厂就自带电话、短信、浏览器一样，你只需要装那些"额外"的Skill。

**第四步：设定检查点。**

不是每个步骤都放行。我在"摘要完成"和"排版完成"之后各设了一个检查点——Agent跑到这里会暂停，等我确认。为什么是这两个点？因为摘要决定了信息的准确性（第7章讲过的教训——AI容易"帮"你编内容），排版决定了对外展示的质量（格式错乱比没做更丢人）。搜什么关键词、怎么发邮件，这些事不需要我干预；但信息的准确性和格式，必须我签字。

**第五步：试跑和迭代。**

第一次跑几乎肯定不完美。我的第一次试跑，Agent搜出了20条结果——其中有8条是软文广告，3条是三个月前的旧闻，有效信息只有9条。问题出在过滤步骤的Prompt不够精确——没告诉Agent"只保留最近7天的内容"和"排除含'赞助''推广'字样的结果"。修改Prompt，再跑一遍，筛到了12条有效结果。又跑一遍，摘要的准确度不够——有几条摘要把"计划融资"写成了"已完成融资"，程度差别很大。加一条约束"融资状态必须精确区分'计划/进行中/已完成'"，再跑，终于稳了。

这就是第5章讲的DMAIC循环——定义、测量、分析、改进、控制。Agent任务也需要迭代。第一次配置只是起点，通过试跑发现问题、修改SOP、再试跑，直到成功率稳定在可接受的范围内。

**关键心法：Agent任务设计 = 给实习生写SOP。**

你好不好用Agent，取决于你写的SOP好不好。SOP模糊，Agent就乱跑；SOP精确，Agent就像一个靠谱的实习生。实习生不会替你思考——但如果你把步骤写清楚了，实习生可以帮你把执行做到60分，你再花20分钟从60分提到80分。比起从0分干到80分的两三个小时，这是巨大的杠杆。
