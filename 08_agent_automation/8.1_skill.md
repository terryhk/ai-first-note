# 8.1 Skill：Agent的"技能包"

在理解Agent怎么自己跑流程之前，先搞清楚一个问题：Agent凭什么能做事？

答案是Skill（技能）。

你买一台新手机，开机能干嘛？打电话、发短信，没了。你想刷微博，装个微博App；你想叫外卖，装个外卖App；你想剪视频，装个剪辑App。手机本身只是个平台，真正让手机"能干"的是你装上去的那些App。

Agent也一样。一个裸的Agent——只有大模型，没有任何Skill——就像一台刚出厂的手机，能聊聊天，但干不了实事。你给它装上"搜索"的Skill，它就能上网查信息；装上"数据分析"的Skill，它就能跑统计；装上"发邮件"的Skill，它就能帮你把结果发出去。

Skill是什么？**一个可装配、可复用、可组合的AI能力模块。** 每个Skill包含两部分：

- 一份说明书（SKILL.md）：告诉AI这个技能做什么、什么时候用、怎么用
- 可选的工具脚本：具体执行操作的代码

你可能已经注意到了，Skill和第4章讲的Function Calling有几分相似——都是让AI调用外部工具。但它们有关键区别：Function Calling是AI被动等待你指令时才调用的，Skill是AI在跑流程时主动检索并激活的。打个比方，Function Calling就像你打开外卖App手动下单，Skill就像你设了一个"每周一中午自动下单同一家店"的定时任务——不需要你每次动手，到时间它自己触发。

Skill有三个核心理念：

**可复用。** 你写了一个"数据分析"Skill，在50个不同的Agent里都能用。就像一个App，你手机装了它，你老婆的手机也能装。写一次，到处跑。

**可组合。** "搜索"Skill +"摘要"Skill +"发邮件"Skill，这三个Skill串起来，就是一个"行业简报"Agent。就像你手机里的地图App、打车App、支付App——三个App组合起来，完成"从公司打车回家并自动付款"这条流水线。单个App只做一件事，组合起来就能做复杂的事。

**可迭代。** 你升级了"搜索"Skill的版本，所有使用这个Skill的Agent自动受益。就像App Store里的App更新了，你手机上那个App也跟着升级，不需要每个用户重新写一遍。这个特性特别重要——你不需要为了改进一个Agent而去改所有Agent，只需要改Skill这一处。

判断原则：**如果你的任务只做一次，不需要Skill；如果你的任务会重复，Skill就是值得投入的标准件。** 回到第7章周报四步法的逻辑——固化模板的投入换取后续每次使用的边际成本趋近于零。Skill是比模板更高一层的标准件：模板固化的是输出格式，Skill固化的是整个能力模块。
