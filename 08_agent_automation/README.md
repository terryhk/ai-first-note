# 第8章 实践3：用Agent自动化重复任务——从ReAct到放手

> **本章心法：** Agent就是"按你写的SOP跑流程的AI实习生"。你定规矩，它跑腿。

---

如果你一路读到了这里，你已经掌握了不少AI武器：契约规格书帮你约束单次对话，思维链帮AI一步步想清楚，RAG帮AI带着知识库回答，PAL帮AI用代码算对数。但你可能还有一个挥之不去的念头——

"每次还是我自己要一步步指挥AI，能不能让它自己跑？"

这个问题让我想起我刚用OpenClaw时的体验。最开始，我只用它来和AI聊天——写写邮件、改改PPT，都是一问一答的模式。直到有一天，我每周要花两三个小时整理"AI行业动态简报"——搜集新闻、筛掉广告和旧闻、写摘要、排版、发邮件——我突然想，这种固定流程的重复任务，能不能让AI自己跑一遍，我只在最后看一眼？

答案是可以。但前提是你要理解一个关键概念：Agent。

"Agent"这个词听起来很唬人，好像是什么超级AI要自己决定一切了。别怕。Agent不是"AI自己干一切"，而是"按你写的SOP跑流程的AI实习生"。你写好SOP，它按SOP跑；你没写SOP，它就跑偏。它的"聪明"由底层模型决定，但它的"能干"完全由你决定。

本章会拆解Agent的三个核心部件——Skill、Harness、ReAct——然后用一个真实的行业简报任务把三个部件串起来，最后讲清楚"人在环"：什么环节放手、什么环节必须抓住。

---

### 一句话心法

**Agent = Skill（技能包）+ Harness（运行时容器）+ ReAct（思考-行动循环）。** 你负责设计SOP，Agent负责跑SOP。你放手的是执行，你不放的是判断。

---

### 🎯 真实场景

我每周都要做一份"AI行业动态简报"。流程是这样的：打开五六个信息源，逐条浏览，筛掉广告和无关内容，剩下的写摘要，按"技术/商业/政策"分类，最后排版成一封邮件发给团队。整个过程大概两三个小时，内容每周不同，但流程每周一样。

这就像你在工厂流水线上拧螺丝——螺丝的种类每周在变，但拧螺丝的动作永远是那个动作。你不会因为换了一批螺丝就不会拧了，工厂也不会因为换了螺丝就重新培训你。流程是固定的，变化的只是原材料。

Agent就是那条流水线。你把流程设计好，每周自动跑一遍。你唯一要做的事，就是最后看一眼成品——数据对不对、摘要准不准、有没有漏掉大事。二十分钟审校，替代两三个小时的手工劳动。

好，接下来我把这条流水线的三个核心部件拆开来讲。
