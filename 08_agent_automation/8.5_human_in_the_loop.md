# 8.5 人在环

"人在环"（Human-in-the-Loop）听起来是个术语，其实就是一句大白话：**Agent跑流程，你在关键步骤审批。**

我在上一节的行业简报例子里已经用到了人在环——摘要完成和排版完成后，Agent暂停等我确认。这就是人在环的最基本形式：**不是每一步都需要你，但关键步骤必须有你。**

为什么要人在环？一句话：**完全放手 = 灾难，有检查点的Agent = 杠杆。**

完全放手是什么画面？你让Agent"帮我写周报并发出去"——不加任何检查点。Agent写完直接发了。结果呢？可能有一周数据口径错了，领导来问；可能有一周风险描述太重，把"关注"写成了"紧急"，全团队紧张了半天；可能有一周漏了一条重要新闻，竞争对手已经发布了新产品你还不知道。出一次问题，你对Agent的信任就会打折；出两次，你就不敢用了；出三次，你回到手工操作——然后Agent变成摆设。

所以人在环不是"不信任Agent"，而是"保护Agent的好声誉"——让Agent只输出经过你确认的东西，它的可靠性才有保障，你才会持续愿意用它。

但人在环也不意味着每一步都要你审批。如果你每步都审批，等于没自动化——你只是把"亲自做"变成了"亲自审批"，效率没提升。正确的方式是：

**判断要不要放手的三个标准：**

1. **出错后果的严重程度。** 邮件签名格式错了，无所谓——放手；客户报价数字错了，后果严重——审批。
2. **AI出错的概率。** 搜索关键词这种事AI几乎不会出错——放手；判断新闻是否属于"技术类"这种分类任务，AI偶尔会错——审批。
3. **纠错成本。** 纠错只要改一个词——放手；纠错要重新跑整个流程——提前审批。

三条标准综合判断，而不是只看一条。一条新闻分类错了（概率中等、后果中等、纠错成本低）——可以放手；报价数字（概率中等、后果严重、纠错成本高）——必须审批。

我把检查点分成三档：

| 检查点等级 | 含义 | 何时使用 |
|-----------|------|---------|
| 自动放行 | Agent直接通过 | 出错后果低 + AI错误率低 + 纠错成本低 |
| 事后抽查 | Agent通过，你随机抽查 | 出错后果中 + AI错误率低 + 纠错成本低 |
| 事前审批 | Agent暂停等你确认 | 出错后果高 或 AI错误率高 或 纠错成本高 |

行业简报的流程里，搜索步骤是自动放行，摘要步骤是事后抽查（我每周抽两条对照原文），排版步骤和发送步骤是事前审批。这样我的介入时间压缩到每周可能只花半小时，但关键质量卡点一个没少。

人的价值不是"做更多事"，而是"在关键节点做判断"。第4章讲过半人马协作——人类判断 + AI执行。Agent把"执行"这部分自动化的程度推到了新高，但"判断"这个核心，永远是你。
