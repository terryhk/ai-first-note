### 3.3 思维链：合同的"分期付款"机制

如果说4S原则是让合同"写得好"，那思维链就是让合同"执行得稳"。

思维链是近年来Prompt工程领域最重要的技术突破。它的核心做法极其简单——简单到你可能会觉得"就这？"——就是在你的Prompt末尾加一句："请一步步思考。"

就这一句话。

#### 为什么有效？

回到第一章的核心：AI是一个概率采样机器。复杂问题直接要答案，相当于让AI在概率空间里做一次长距离跳跃——从一个复杂问题直接跳到最终答案。这个跳跃的跨度太大，中间经过的每一步都有可能偏到岔路上去，最终到达正确位置的概率很低。

思维链做的是什么呢？它把一次长距离跳跃拆成了多次短距离跳跃。每次只跳一小步——从前一个推理步骤到下一个推理步骤——每一步的正确概率都远高于直接跳总距离的正确概率。

用数学直觉来理解（不列公式）：你让AI直接从问题跳到答案，那是在赌一次概率。你把推理过程拆成一段一段，每一步都是在赌一个更小的概率空间——每一步的赌注更小，每一步赢的概率更高。所有小步都赢的总概率，远比一步跳赢的的概率高得多。

我把它叫做"分期付款"机制。总包合同——AI一次交付全部成果——风险集中在你身上。分期付款——每走一步付一笔、结一次账——每一次都在降低违约风险。

#### 一个案例：苹果加减

给你两个Prompt，你自己感受区别：

**Prompt A（无思维链）：**
```
问题：我有5个苹果，吃了2个，朋友给了我3个，我又给了朋友1个，我朋友接着又吃了我2个苹果，现在我有几个苹果？
```

AI直接输出：5 - 2 + 3 - 1 - 2 = 3。答案是3个。

这个答案对吗？不对。因为最后一句话是"我朋友接着又吃了我2个苹果"——注意苹果是从我这里被吃的，不是朋友自己吃自己的苹果。所以应该是5 - 2 + 3 - 1 - 2 = 3。等等——让我再算一遍。初始5个，吃了2个=剩3个，朋友给了3个=有6个，给了朋友1个=剩5个，朋友吃了我的2个=剩3个。答案是3个。看起来对。

但如果我稍微改一下问题呢？

**Prompt B（有思维链）：**
```
问题：我有5个苹果，吃了2个，朋友给了我3个，我又给了朋友1个，我朋友接着又吃了我2个苹果，现在我有几个苹果？

请一步步思考，每一步都列出当前剩余数量。
```

AI输出：
1. 一开始我有5个苹果。
2. 我吃了2个，剩余：5 - 2 = 3个。
3. 朋友给了我3个，剩余：3 + 3 = 6个。
4. 我给了朋友1个，剩余：6 - 1 = 5个。
5. 朋友吃了我的2个，剩余：5 - 2 = 3个。

最终答案：3个苹果。

这个例子里，两个答案一样——因为题目本身不复杂。但当问题变复杂时，思维链的价值就出来了。比如你加一个转弯："朋友吃了我的2个苹果，但我发现他吃的那2个里，有1个其实是他早上给我的。"——这个问题用思维链跑，AI会清晰追踪每一步的归属变化。没有思维链，AI大概率会直接跳到一个错误答案，因为它在概率空间里一次跳过了多个转换关系，把"谁的苹果"搞混了。

#### 思维链的三种用法

这不是"三种模板"让你去套——它们是在不同场景下选择的不同策略。

**第一种：零样本思维链。** 最简单。就是在Prompt末尾加一句"请一步步思考"或者"Let's think step by step"。不需要给任何示例，不需要做任何特殊设置。就这么一句话——你的Prompt质量就能提升一个档次。适用于大多数需要推理的任务。

**第二种：少样本思维链。** 给出2-3个示例，每个示例完整展示从问题到推理到答案的全过程。AI从这些示例中学习"推理模式"，然后应用到你的新问题上。适用于需要特定推理路径的任务——比如法律分析、财务建模、逻辑推理，这些领域有自己特定的推理方式，零样本的"一步步思考"不够好，你需要给AI示范"这种问题就应该这样推理"。

**第三种：分步思维链。** 不依赖AI自己拆步骤——你替它拆。把一个大问题拆成3-5个连续的小问题，逐个让AI回答，前一个问题的答案作为后一个问题的上下文。这种方式对人来说比较累，但效果往往更好——因为拆步骤这个工作本身需要判断力，而判断力恰恰是人的第五维优势。

举个例子，不写"帮我分析这家公司的投资价值"——而是写：
1. "先分析这家公司的收入结构和增长趋势"
2. "基于第一步的分析，再看它的利润率和现金流"
3. "结合前两步，评估它的估值水平和行业对比"
4. "最后，综合考虑所有因素，给出投资建议"

每一个问题都专注、清晰、范围更窄。AI在每个问题上的成功率都远高于那个大问题。

> **概率工具箱**
>
> 思维链的本质是强迫AI缩小每次推理的跳跃距离。直接要答案等于一次长距离跳跃——跨越整个推理链条，一步到位，P可能只有20%-40%（取决于问题复杂度）。用思维链拆成一步步，每一小步的P都在80%-90%以上，五步总成功概率大约是0.85^5≈44%——乍看不高，但这是最保守的估算。实际上，由于每步都在上一步的基础上缩小了搜索空间，累积P会更高。而且你可以在每一步审查AI的输出——发现偏了就纠正——这个迭代过程会把最终P推得更高。
